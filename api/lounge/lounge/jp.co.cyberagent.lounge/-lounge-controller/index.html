LoungeController

abstract class LoungeController(lifecycle: Lifecycle, modelBuildingDispatcher: CoroutineDispatcher) : LoungeBuildModelScope, AutoCloseable
Content copied to clipboard

A controller for easily combining LoungeModel instances in a ObjectAdapter. The implementation is inspired by epoxy/EpoxyController.

Simply override buildModels to declare which models should be used, and in which order. Call requestModelBuild whenever your data changes, and the controller will call buildModels, update the adapter with the new models, and notify any changes between the new and old models.

The controller maintains an ObjectAdapter with the latest models, which you can get via adapter.

If you only need to build models once, you can use objectAdapterWithLoungeModels which will construct an ObjectAdapter directly. If you prefer composition instead of inheriting LoungeController, you can use LambdaLoungeController to implement buildModels via a lambda function.

All data change notifications are applied automatically via androidx.recyclerview.widget.DiffUtil's diffing algorithm. All of your models must have a unique LoungeModel.key set on them for diffing to work.
